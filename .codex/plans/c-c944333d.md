# Codex plan for mehrangta/funny-whiteboard-paint#1

Job: `c-c944333d` · Revision: 2

Add a frontend-only Text tool with the authorized behavior: users select Text, click the whiteboard, type multiple lines in an inline editor, apply one color/size/bold/italic style to the entire text block, press Ctrl/Cmd+Enter to rasterize it into the canvas, or press Escape to cancel it. Committed text becomes part of the existing PNG bitmap and is changed later by undoing and retyping. The implementation requires no Rust command, dependency, capability, migration, or saved-file-format changes.

## Validation

- Install the absent dependencies with the README-defined `bun install`, then run the package-defined `bun run build` to validate strict TypeScript checks and the Vite production build.
- Run `bun run dev` for frontend-only visual and interaction checks. Canvas text editing and rendering can be exercised there, but Rust-backed history, save/load, and runtime mascot calls are expected to warn and return null outside Tauri.
- Run the repository-defined `bun run tauri dev` for full integration validation. Confirm plain, bold, italic, and bold-italic text render with the selected color at 8 px, 32 px, and 96 px.
- Verify Enter creates explicit new lines, Ctrl+Enter and Cmd+Enter commit one text block as exactly one history entry, Escape cancels without changing the canvas or history, whitespace-only input creates no snapshot, and IME composition is not interrupted by the commit shortcut.
- Verify formatting controls update the entire pending block without committing it; switching to another tool commits non-empty text, clicking a new board position commits the old block before opening the next editor, and clicking the active Text tool leaves the current editor open.
- Verify the first Undo or Redo click during an uncommitted session only cancels that session. After commit, Undo removes the complete text block in one action and Redo restores it. Confirm the existing brush, line, rectangle, clear, and history behavior remains unchanged when no editor is active.
- Verify Save commits pending non-empty text before PNG export. Load and Clear discard pending text before replacing the board. Save a formatted multiline drawing, clear it, reload it, and confirm pixels, transparency, positioning, color, size, bold, and italic appearance survive the PNG round trip.
- Resize the Tauri window while the editor is open and immediately before committing. Confirm the editor remains anchored and clipped to the board, typed content is preserved, delayed bitmap restoration does not erase committed text, and resize creates no additional history entry. Repeat after text is committed to confirm it follows existing bitmap scaling at normal and high device-pixel ratios.

## Risks

- Committed text is intentionally rasterized and cannot be selected, moved, or restyled. Editable placed text would require a retained object model and a project format beyond the current PNG-only design.
- DOM textarea and Canvas 2D font metrics may vary slightly between Windows WebView2 and browser-only development. Using the same explicit system font stack, size, weight, style, and 1.2 line height minimizes but may not eliminate platform-specific glyph differences.
- The existing toolbar is already space-constrained and overlays the scene. Adding Text and formatting controls can obscure the drawing area or overflow unless all three existing responsive ranges are manually validated.
- The current resize implementation restores a data URL asynchronously. Without the specified shared resize-completion sequencing, a text commit made during resize could be overwritten by the delayed restoration.
- The Rust `push_state` command does not clear redo history when a new snapshot is created after undo. Text uses the existing history implementation; changing that pre-existing behavior is outside this issue, so branched-history behavior may remain inconsistent.
- The repository contains no automated test, lint, browser integration, or Rust test suite. Pointer, keyboard, IME, responsive layout, resize timing, and Tauri persistence regressions therefore require the listed manual checks.
